# 文本直查快捷键设计

**日期：** 2026-04-24

## 目标
为 Windows 版增加“先选中文字，再按快捷键直接查询”的能力，避免不必要的截图、图片上传和额外 token 消耗。

## 方案
- 新增独立快捷键：`textQueryShortcut`
- 触发后读取当前选中文本：保存剪贴板、模拟复制、读取文本、恢复剪贴板
- 成功时直接打开聊天面板并自动发送：`请简洁的解释这段文本：\n\n<选中文本>`
- 失败时提示：`未检测到可查询文本，请改用截图查询`

## 快捷键决策
- 文本直查：`Ctrl+Shift+A`
- AI 截图：默认迁移为 `Ctrl+Shift+Q`
- 普通截图：`Ctrl+Shift+S`

这样可以满足用户把 `Ctrl+Shift+A` 专门留给文本直查，同时避免与现有 AI 截图快捷键冲突。

## 范围
- 仅做 Windows
- 不做按住后拖选文本模式
- 不新增可配置的文本查询提示词，先按用户指定文案写死
